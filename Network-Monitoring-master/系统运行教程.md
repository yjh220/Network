# 网络入侵检测与防御系统 - 运行教程

## 一、环境准备

### 1.1 硬件要求

| 配置 | 最低要求 | 推荐配置 |
|-----|---------|---------|
| CPU | 双核2.0GHz | 四核3.0GHz+ |
| 内存 | 4GB | 8GB+ |
| 硬盘 | 10GB可用空间 | 50GB+ SSD |
| 网卡 | 任意以太网卡 | 支持混杂模式的网卡 |

| 软件 | 版本要求 | 说明 |

## 二、安装步骤

### 2.1 下载项目

**方式一：直接下载ZIP**
1. 访问项目地址
2. 点击"Download ZIP"下载
3. 解压到本地目录

**方式二：Git克隆**
```bash
git clone [项目地址]
cd Network-Monitoring-master/Network-Monitoring-master/Network-Monitoring-master
```

### 2.2 创建Python虚拟环境（推荐）

**Windows:**
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 看到命令行前缀出现 (venv) 表示激活成功
```

**Linux/macOS:**
```bash
# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate
```

### 2.3 安装Python依赖

```bash
# 进入项目目录
cd C:\Users\29236\Desktop\Network-Monitoring-master\Network-Monitoring-master\Network-Monitoring-master

# 安装所有依赖
pip install -r requirements.txt
```

**如果安装速度慢，可使用国内镜像源：**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**依赖安装验证：**
```bash
# 验证关键依赖是否安装成功
python -c "import flask, scapy, sklearn; print('依赖安装成功！')"
```

### 2.4 安装WinPcap（Windows用户）

1. 下载WinPcap：https://www.winpcap.org/install/
2. 运行安装程序，按默认选项安装
3. 安装完成后重启计算机

**Windows 10/11用户注意：**
如果WinPcap安装失败，可使用Npcap替代：
- 下载地址：https://nmap.org/npcap/
- 安装时勾选"Support raw 802.11 traffic"和"Install Npcap in WinPcap API-compatible Mode"

## 四、详细功能使用

### 4.1 用户登录与注册

#### 首次登录
1. 访问 http://127.0.0.1:8080
2. 点击右上角"登录"按钮
3. 输入默认管理员账户：
   - 用户名：`admin`
   - 密码：`admin123`
4. 点击"登录"

#### 注册新用户
1. 访问 http://127.0.0.1:8080/register
2. 填写注册信息：
   - 用户名：3-20个字符，字母数字下划线
   - 邮箱：有效邮箱地址
   - 密码：至少6个字符
   - 确认密码
3. 点击"注册"按钮
4. 注册成功后自动登录

### 4.2 态势仪表板使用

#### 访问仪表板
1. 登录后，点击顶部导航"仪表板"
2. 或直接访问：http://127.0.0.1:8080/dashboard

#### 仪表板功能
- **实时流量图表**：展示入站/出站流量趋势
- **协议分布**：饼图显示TCP/UDP/HTTP等协议占比
- **威胁统计**：今日检测到的威胁数量
- **活跃IP列表**：当前网络中的活跃主机
- **IP快速封禁**：点击IP右侧的"封禁"按钮

### 4.3 网络监控使用

#### 访问网络监控
1. 点击顶部导航"网络监控"
2. 或访问：http://127.0.0.1:8080/network_monitor

#### 网络监控功能

**启动实时监控：**
```bash
# 在主界面点击"启动系统"按钮
# 系统将开始捕获网络流量
```

**数据包查看：**
- 实时显示捕获的数据包列表
- 点击数据包查看详细信息
- 支持十六进制数据查看

**协议统计：**
- 实时显示各协议占比
- 包括TCP、UDP、ICMP、HTTP等

### 4.4 入侵检测使用

#### 访问入侵检测
1. 点击顶部导航"入侵检测"
2. 或访问：http://127.0.0.1:8080/intrusion_detection

#### 威胁检测步骤

**方式一：PCAP文件分析**
1. 准备一个.pcap格式的网络包文件
2. 点击"选择文件"按钮
3. 选择PCAP文件
4. 点击"开始分析"
5. 等待分析完成，查看威胁结果

**方式二：CSV特征文件分析**
1. 准备流量特征CSV文件
2. 点击"选择文件"按钮
3. 选择CSV文件
4. 点击"开始分析"
5. 查看威胁识别结果

**实时威胁检测：**
1. 在主界面启动系统
2. 系统自动捕获流量
3. 点击"停止系统"进行威胁分析
4. 查看检测结果

### 4.5 IP封禁与解封

#### 手动封禁IP
1. 在仪表板的活跃IP列表中
2. 找到要封禁的IP
3. 点击右侧"封禁"按钮
4. 确认封禁操作

#### 解封IP
1. 点击顶部导航"设置"
2. 在"IP黑名单"中找到已封禁IP
3. 点击"解封"按钮

### 4.6 后台管理使用

#### 访问后台管理
1. 确保以管理员身份登录
2. 访问：http://127.0.0.1:8080/admin
3. 或点击顶部"后台管理"链接

#### 用户管理
1. 进入"用户管理"页面
2. 查看、创建、删除用户
3. 修改用户角色（普通用户/管理员）
4. 启用/禁用用户账户

#### 防御规则管理
1. 进入"防御策略"页面
2. 查看当前防御规则
3. 添加新的防御规则
4. 启用/禁用规则

#### 系统日志查看
1. 进入"系统日志"页面
2. 查看登录日志
3. 查看操作日志
4. 按时间/用户筛选

## 六、常见问题排查

### 6.1 启动失败

**问题：端口被占用**
```
OSError: [Errno 48] Address already in use
```
**解决方案：**
1. 关闭占用8080端口的程序
2. 或修改app2.py中的端口号

**问题：依赖缺失**
```
ModuleNotFoundError: No module named 'xxx'
```
**解决方案：**
```bash
# 重新安装依赖
pip install -r requirements.txt
```

### 6.2 数据包捕获失败

**问题：WinPcap未安装**

**解决方案：**
1. 下载并安装WinPcap或Npcap
2. 重启计算机

**问题：权限不足**

**解决方案：**
- Windows：以管理员身份运行
- Linux：使用sudo运行

### 6.3 异常检测模块不可用

**问题：模型文件不存在**

**解决方案：**
1. 检查 `model/` 目录下是否有：
   - `randomforest_model.pkl`
   - `preprocessing_pipeline.pkl`
2. 如果没有，联系项目提供者获取模型文件

### 6.4 登录问题

**问题：忘记管理员密码**

**解决方案：**
```bash
# 运行数据库初始化脚本
python init_admin_db.py
# 将重置admin密码为 admin123
```
