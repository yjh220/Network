# 登录问题调试指南

## 问题：登录/注册后无法跳转到主页

### 解决步骤：

#### 1. 检查数据库是否已初始化

运行以下命令检查：
```bash
python test_login.py
```

如果提示"数据库中没有用户"，请运行：
```bash
python init_db.py
```

#### 2. 检查数据库文件

确认 `network_monitor.db` 文件存在于项目根目录。

#### 3. 测试登录功能

使用默认账户测试：
- 用户名: `admin`
- 密码: `admin123`

#### 4. 清除浏览器缓存

按 `Ctrl+Shift+Delete`，清除：
- 缓存的图片和文件
- Cookie和其他网站数据

#### 5. 检查浏览器控制台

按 `F12` 打开开发者工具，查看：
- **Console** 标签：查看是否有JavaScript错误
- **Network** 标签：查看登录请求是否成功发送

#### 6. 检查应用日志

启动应用后，查看控制台输出是否有错误信息。

### 常见问题及解决方法：

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 提示"用户名或密码错误" | 数据库没有用户 | 运行 `python init_db.py` |
| 点击登录没有反应 | JavaScript错误 | 清除浏览器缓存，检查控制台 |
| 登录后跳转失败 | 路由配置问题 | 确保使用最新版本的app2.py |

### 完整重置步骤：

如果以上方法都不行，执行完整重置：

1. 停止应用

2. 删除数据库文件：
```bash
del network_monitor.db
```

3. 重新初始化：
```bash
python init_db.py
```

4. 重新启动应用：
```bash
python app2.py
```

5. 清除浏览器缓存后重试
