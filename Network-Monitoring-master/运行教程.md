# 网络入侵检测与防御系统 - 运行教程

## 系统概述

这是一个基于机器学习的网络入侵检测与防御系统，具有以下功能：
- 实时网络监控
- 威胁流量检测（支持7种威胁类型）
- IP地址阻止/解除阻止
- 流量可视化分析
- 攻击演示功能

## 三、功能使用指南

### 3.1 主界面 (首页)
**访问地址：** http://127.0.0.1:8080/

- 查看系统概览
- 导航到各个功能模块
- 查看实时系统状态

### 3.2 仪表板
**访问地址：** http://127.0.0.1:8080/dashboard

**功能：**
- 查看实时网络流量统计
- 监控活跃IP连接
- **阻止可疑IP地址**
- 查看威胁检测统计

**如何阻止IP：**
1. 在"活跃IP"表格中找到要阻止的IP
2. 点击该IP行右侧的"阻止"按钮
3. IP将被添加到阻止列表，状态变为"已阻止"
4. 点击"解除阻止"可以恢复该IP

### 3.3 网络监控
**访问地址：** http://127.0.0.1:8080/network_monitor

**功能：**
- 查看实时流量协议分布
- 监控网络数据包
- 查看连接状态

**如何使用：**
1. 点击"启动系统"开始监控
2. 系统会显示实时流量数据（模拟数据）
3. 可以查看协议分布图表

## 五、常见问题

### Q1: 启动时出现 "WARNING: No libpcap provider available"
**A:** 这是正常的警告，表示没有管理员权限发送真实网络包。系统会使用模拟数据，不影响功能使用。

### Q2: 威胁检测结果显示全是"正常流量"
**A:** 系统已启用演示模式，当检测到全部为正常流量时，会自动添加演示威胁数据。请确保：
1. 先点击"分析文件"
2. 再点击"开始威胁检测"

### Q3: 点击"阻止"按钮显示服务器异常
**A:** 请确保使用的是 `app2.py` 启动的系统，该版本包含IP阻止功能。

### Q4: 无法访问网页
**A:** 检查：
1. Python进程是否正在运行
2. 端口8080是否被占用
3. 防火墙是否阻止了连接

### Q5: 模型版本警告
**A:** sklearn版本不匹配的警告可以忽略，不影响功能使用。
